# Java异常处理

## 一、前言

在任何一个稳定的程序中，都会有大量的代码在处理错误，有一些业务错误，我们可以通过主动检查判断来规避，可对于一些不能主动判断的错误，例如 RuntimeException，我们就需要使用 try-catch-finally 语句了。

有人说，错误处理并不难啊，try-catch-finally 一把梭，try 放功能代码，在 catch 中捕获异常、处理异常，finally 中写那些无论是否发生异常，都要执行的代码，这很简单啊。

处理错误的代码，确实并不难写，可是想把错误处理写好，也并不是一件容易的事情。

接下来我们就从实现到 JVM 原理，讲清楚 Java 的异常处理。

## 二、Java 异常处理

### 2.1 概述

既然是异常处理，肯定是区分异常发生和捕获、处理异常，这也正是组成异常处理的两大要素。

在 Java 中，抛出的异常可以分为显示异常和隐式异常，这种区分主要来自抛出异常的主体是什么，显示和隐式也是站在应用程序的视角来区分的。

- **显示异常**：应用程序使用 `throw` 关键字主动将异常实例抛出
- **隐式异常**：Java 虚拟机执行过程中遇到无法继续执行的异常状态

### 2.2 使用异常捕获

在我们编写异常处理代码的时候，主要就是使用 try-catch-finally 这三种代码块：

```java
try {
    // 包含待监控异常的代码
} catch (ExceptionType1 e) {
    // 处理异常
} catch (ExceptionType2 e) {
    // 处理另一种异常
} finally {
    // 必定会运行的代码
}
```

### 2.3 异常的类型

在 Java 中，所有的异常都是 Throwable 类或其子类的实例：

- **Exception**：表示程序可能需要捕获并且处理的异常
- **Error**：表示当触发 Error 时，执行状态已经无法恢复，需要中止线程甚至是中止虚拟机

### 2.4 性能问题

在没有发生异常的情况下，try-catch 对性能的影响微乎其微。但是一旦发生异常，性能差别会非常明显。因此，我们应该尽可能的避免通过异常来处理正常的逻辑检查。

## 三、JVM 如何处理异常

当异常发生时，异常实例的构建是非常消耗性能的。这是由于在构造异常实例时，Java 虚拟机需要生成该异常的异常栈（stack trace）。

当把 Java 代码编译成字节码后，每个方法都会附带一个**异常表**，其中记录了当前方法的异常处理信息。

## 总结

- 异常处理是 Java 编程中不可或缺的一部分
- 要避免使用异常来处理正常业务逻辑
- 理解 JVM 异常处理原理有助于编写更好的代码
- finally 块中的 return 语句会阻止异常的正常抛出
